# 选做一：信号滤波 — 文档说明

> **芯片**: STM32F103C8T6  
> **信号源**: 10kΩ 电位器分压 (PA1, ADC1_IN1, 0~3.3V)  
> **滤波对象**: 1 路模拟电压信号 (CH1: PA1)  
> **日期**: 2026-06-17

---

## 一、滤波方案设计

### 1.1 三层信号处理架构

```
电位器 → ADC采样(12bit) → DMA循环缓冲(32半字)
                              │
              ┌───────────────┼───────────────┐
              ▼               ▼               ▼
         raw (单次)      avg (均值)       ema (指数)
         无滤波           16-sample       α=0.15 EMA
         最大抖动         中等平滑        最强平滑
```

| 层级 | 函数 | 数据处理 | 响应速度 | 平滑度 |
|------|------|---------|---------|--------|
| **raw** | `ADC_DMA_ReadRaw(0)` | DMA 缓冲区第 1 个采样点，不做任何处理 | 瞬时响应 | 差（受量化噪声影响） |
| **avg** | `ADC_DMA_Read()` | 16 个连续采样点的算术平均 | 微秒级（硬件 DMA 连续采样） | 中（抑制高频随机噪声） |
| **ema** | `ADC_EMA_Filter()` | 指数移动平均，α=0.15 | 秒级（需多个周期收敛） | 强（对突变和毛刺高度免疫） |

### 1.2 EMA 滤波算法

```
filtered[n] = α × raw[n] + (1 - α) × filtered[n-1]

其中:
  α = 0.15 (新采样权重 15%，历史权重 85%)
  raw[n] = 当前 16-sample 均值电压
  filtered[n-1] = 上一次 EMA 输出
```

**α 选取原则**：
- α 越大 → 响应越快、平滑越弱
- α 越小 → 响应越慢、平滑越强
- 0.15 适合缓慢变化的直流信号（电位器手动调节场景）

**收敛时间估算**：EMA 达到稳态值 63.2% 需要 τ = (1-α)/α = 5.67 步。本系统 EMA 每 500ms 更新一次（在 5s 打印中调用），达到 95% 收敛约需 3τ ≈ 17 步 ≈ 8.5 秒。

## 三、差异分析与讨论

### 3.1 raw vs avg — 为何差异不明显？

从数据可见，在本次测试中 `raw` 和 `avg` 几乎完全一致（差值 ≤0.02V）。原因：

1. **ADC DMA 采样速率极高**：ADC 时钟 12MHz，16 个采样在约 10μs 内完成，远小于 5s 的打印间隔。在 5s 尺度上，16-sample 均值等效于瞬时值。
2. **电位器为直流信号**：无高频噪声源（如电机、开关电源），ADC 输入本身波动极小。
3. **硬件去耦良好**：STM32 最小系统板自带滤波电容。

**raw 和 avg 的差异会在以下场景中显现**：
- 电机运行时产生电磁干扰
- 电源纹波较大时
- ADC 输入引脚悬空或接触不良时
- 采样速率降低至 Hz 级别时

**如何验证 avg 的降噪效果**：缩短打印周期至 100ms，或在电位器两端并联小电容引入充放电延迟，可观察到 raw 的逐次跳动 vs avg 的平稳。

### 3.2 avg vs ema — 惯性平滑效果显著

这是本次测试中最明显的差异，验证了 EMA 滤波器的核心特性：

| 现象 | 数据证据 | 解释 |
|------|---------|------|
| **上升沿滞后** | t=45s: avg=2.72V, ema=1.69V (差 1.03V) | EMA 历史权重 85%，对快速上升响应迟钝 |
| **下降沿惯性** | t=50s: avg=2.02V (已回落), ema=1.74V (仍在上升) | ema 还未追上峰值就遇到输入回落，出现"过冲感" |
| **稳态缓慢收敛** | t=50s→65s: avg 稳定 2.02V, ema 从 1.74→1.85V | 15s 仅收敛 0.11V，完全收敛需更长时间 |

**EMA 的优势场景**：
- **抑制偶发毛刺**：单次 ADC 采样受 EMI 尖峰干扰读到一个异常高/低值时，EMA 几乎不受影响（单次仅贡献 15% 权重）
- **判断稳态**：当 `|raw - ema|` 小于阈值时，可认为信号已稳定，适合触发后续动作（如确认电位器已拧到位）
- **去抖动**：机械电位器接触不良导致的短暂跳变可被 EMA 有效滤除

**EMA 的劣势场景**：
- **快速变化信号**：若输入变化频率 > 1Hz，EMA 会产生明显相位滞后
- **阈值报警**：依赖 EMA 做阈值判断会导致报警延迟（如电压突变到危险值时 EMA 需要数秒才能反映）

### 3.3 工程应用建议

| 场景 | 推荐滤波方式 | 原因 |
|------|------------|------|
| **实时电压显示** | avg (16-sample 均值) | 响应快、噪声适中，用户观感流畅 |
| **阈值报警** | avg (16-sample 均值) | 不能有滞后，安全相关必须快速响应 |
| **稳态判断/数据记录** | ema (α=0.15) | 去抖动、确定最终稳定值 |
| **存在强 EMI 环境** | raw + 中值滤波 或 滑动窗口滤波 | 均值对脉冲干扰敏感，中值滤波更鲁棒 |

## 四、代码实现摘要

### 4.1 新增 API (adc_dma.h)

```c
/* 读取单次原始 ADC 值（无滤波，用于对比） */
uint16_t ADC_DMA_ReadRaw(uint8_t ch);

/* 初始化 EMA 滤波器 */
void ADC_EMA_Init(void);

/* 指数移动平均滤波 */
float ADC_EMA_Filter(float raw_voltage, uint8_t ch);
```

### 4.2 滤波对比输出 (main.c — 每 5s)

```c
uint16_t raw_single = ADC_DMA_ReadRaw(0);       // 单次原始采样
float    raw_v      = ADC_ToVoltage(raw_single); // 无滤波电压
float    mean_v     = adc.ch1.voltage;           // 16-sample 均值
float    ema_v      = ADC_EMA_Filter(mean_v, 0); // EMA 滤波

printf("CH1 raw=%u.%02uV avg=%u.%02uV ema=%u.%02uV\r\n",
       (unsigned int)raw_v, (unsigned int)(raw_v * 100) % 100,
       (unsigned int)mean_v, (unsigned int)(mean_v * 100) % 100,
       (unsigned int)ema_v, (unsigned int)(ema_v * 100) % 100);
```

### 4.3 关键配置宏

```c
#define ADC_SAMPLE_PER_CH  16       // 每通道 DMA 采样数（均值窗口）
#define ADC_EMA_ALPHA       0.15f   // EMA 平滑系数（越小越平滑）
```

## 五、结论

1. **16-sample 均值滤波**（avg）在硬件层面通过 DMA 完成，能在微秒级消除 ADC 量化噪声，是嵌入式系统中最常用的实时滤波手段。
2. **EMA 指数移动平均**（ema）通过软件实现，α=0.15 提供了显著的惯性平滑效果——对缓慢变化的直流信号（如电位器）可有效消除抖动，但对快速变化的响应存在明显滞后。
3. 本系统中，**阈值报警应继续使用 avg 值**以保证响应速度，**EMA 值可用于判定信号稳态**或作为数据记录时的去抖动参考。
4. 若需在强噪声环境下验证 raw 与 avg 的差异，可在电机/逆变器运行条件下进行对比测试。
